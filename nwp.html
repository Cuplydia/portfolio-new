The Challenge

The challenge of the responsive website using Bootstrap. The goal was to create a widget to include in the not only client's power tools, but appliance hosted sites. Also, to provide an opportunity making customers aware of additional Bosch products. Widget is implemented as a web widget or web part-independent small web page which acts as a section of a main web page.

Planning & Discovery

In the planning and analysis phase, my goal was to figure out who would be using the carousel, what these users wanted / needed, and what features we should we build. After researching, planning, and reviewing other client's websites, we created a list of features and enhancements, which included:

  • Clarifying their message (who they are and what they do)
  • Conveying a feeling of power, excellence, and excitement
  • Highlighting the appliance features
  • Showcasing the type of “Applicances”
  • Telling the story / benefit purchasing applicances

Methods

We analyzed the existing Power Tool carousel and similar ones -
like Home Depot - to determine what was working and what wasn't.
Also looked at existing marketing materials (e.g. website, videos, advertisements, media, etc.) Interviewed potential internal customers - got the client's input regarding what they were looking for, look and feel, timeline, communication style, etc.

Methods

  • Creating an interview guide
  • Organizing and conducting needs assessment and feedback meetings, asking follow-up / clarifying questions, and
  • Note-taking
  • Debriefing / compiling the findings from the interview with the team.

Defining Requirements

From our internet research / client interview, we create a list of requirements (which are high-level tasks that the software should support). These included: We learned to use Bootstrap and Visual Studio to code (lol) Looked at Microsoft SQL database transfer to Visual Studio; Looked at Asp.Net to reviewing file in local host. Looked at what we liked / didn't like. Managed coding task. Don't try and do everything you want to do - do their requirements first:
